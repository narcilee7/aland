## 二、后端架构（Go）

### 2.1 核心模块

#### `core/land.go` — 大陆状态机
```go
type Land struct {
    Tribes    map[string]*Tribe   // 所有部落
    Forge     *Forge              // Token 熔炉
    Eye       *Eye                // 灵动岛状态
    mu        sync.RWMutex
}

type Tribe struct {
    ID        string        // "claude" | "cursor" | "trae" | ...
    Name      string        // 显示名
    Eco       string        // "classical" | "modern" | "oriental" | ...
    Status    TribeStatus   // idle | running | busy | error
    Vital     VitalSign     // 生命体征
    Config    ConfigDNA     // 配置双螺旋
    Sessions  []SessionShard // 记忆碎片
    Adapter   TribeAdapter  // 适配器接口
}

type VitalSign struct {
    PID       int
    CPU       float64       // 0-100
    Memory    float64       // MB
    CWD       string
    Uptime    time.Duration
    Model     string        // 当前使用模型
    UpdatedAt time.Time
}
```

#### `core/forge.go` — Token 熔炉
```go
type Forge struct {
    DailyBudget   int64         // 每日预算 (Token 数)
    TodaySpent    int64
    ByTribe       map[string]int64
    ByModel       map[string]int64
    History       []CostPoint   // 时序数据，用于趋势图
}

type CostPoint struct {
    Timestamp int64
    Tribe     string
    Model     string
    Tokens    int64
    CostUSD   float64
}
```

#### `core/eye.go` — 灵动岛
```go
type EyeState struct {
    Mode      EyeMode    // dormant | active | storm | alert
    Running   []string   // 当前运行的部落 ID
    FlashMsg  *FlashMsg  // 临时通知
}

type FlashMsg struct {
    Type      string     // "complete" | "cost_alert" | "error"
    Tribe     string
    Content   string
    Duration  time.Duration
}
```

### 2.2 部落适配器接口（关键扩展点）

```go
// TribeAdapter 是每个 CLI 的统一抽象
type TribeAdapter interface {
    // 身份
    ID() string
    Name() string
    EcoType() string // 决定前端渲染的地形类型
    
    // 进程
    DetectProcess() (*ProcessInfo, error)  // 扫描是否运行
    Launch(cwd string, args []string) error // 启动
    Kill(pid int) error                      // 终止
    
    // 配置
    ConfigPaths() []string                   // 配置文件路径列表
    ParseConfig() (map[string]interface{}, error)
    WriteConfig(data map[string]interface{}) error
    
    // 会话/日志
    SessionPath() string                     // 会话存储目录
    ParseSessions() ([]SessionShard, error)
    ParseTokenUsage() (*TokenUsage, error)  // 解析 Token 消耗
    
    // 前端元数据
    ThemeColor() string                      // 主色 #d4a853
    AccentColor() string                     // 荧光 accent
}
```

#### 已规划适配器

| 适配器 | 进程检测 | 配置读取 | Token 解析 | 优先级 |
|--------|---------|---------|-----------|--------|
| `ClaudeAdapter` | `ps aux \| grep claude` | `~/.claude/settings.json` | `~/.claude/projects/*/usage.json` | P0 |
| `CursorAdapter` | `ps aux \| grep Cursor` | `~/.cursor/settings.json` | 日志估算 | P1 |
| `TraeAdapter` | `ps aux \| grep Trae` | `~/.trae/config.json` | 日志估算 | P1 |
| `KimiAdapter` | `ps aux \| grep kimi` | `~/.kimi/config.json` | 待确认 | P1 |
| `CodexAdapter` | `ps aux \| grep codex` | `~/.codex/config.json` | `~/.codex/sessions/` | P2 |
| `OpenCodeAdapter` | `ps aux \| grep opencode` | `~/.config/opencode/` | 待确认 | P2 |

**扩展新 CLI**：只需实现 `TribeAdapter` 接口，注册到 `Land.Tribes`，前端自动渲染新地形。

### 2.3 基础设施

#### `infra/proc.go` — 进程管理
```go
type ProcManager struct {
    scanInterval time.Duration // 2s
    callbacks    map[string]func([]ProcessInfo)
}

type ProcessInfo struct {
    PID       int
    Name      string
    CmdLine   string
    CWD       string
    CPU       float64
    Memory    float64
    StartTime time.Time
}
```
- 跨平台实现：macOS 用 `ps` + `lsof`，Windows 用 WMI，Linux 用 `/proc`
- 提供 `Watch(tribeID string, callback)` 订阅特定进程变化

#### `infra/fs.go` — 文件监听
```go
type FSWatch struct {
    watcher *fsnotify.Watcher
    handlers map[string]func(event fsnotify.Event)
}
```
- 监听各 CLI 配置目录，变更时通过 Wails Event 推送到前端
- 防抖处理（500ms）

#### `infra/logparser.go` — 日志解析
```go
type LogParser interface {
    ParseTokenUsage(reader io.Reader) (*TokenUsage, error)
}

// 各适配器可注入自己的 Parser
type ClaudeLogParser struct{}
type CursorLogParser struct{} // 基于输出估算
```

#### `infra/store.go` — 本地持久化
```go
type Store struct {
    db *bbolt.DB // 或 SQLite
}
```
- 存储：预算设置、历史成本、用户偏好、会话索引
- **不存储敏感信息**（API Key 只读实时配置，不落库）

### 3.2 Canvas 引擎（等距投影）

```typescript
// canvas/IsometricEngine.ts
class IsometricEngine {
  private ctx: CanvasRenderingContext2D;
  private camera: Camera;
  private particles: Particle[];
  private tribes: TribeSprite[];
  
  // 等距投影公式
  isoToScreen(x: number, y: number, z: number): Point;
  screenToIso(screenX: number, screenY: number): Point;
  
  // 渲染循环
  render(): void {
    this.clear();
    this.drawBackground(); // 星云粒子
    this.drawTerrain();    // 地形起伏
    this.drawPathways();   // 部落间光路
    this.drawTribes();     // 部落精灵
    this.drawForge();      // 中央熔炉
    this.drawWeather();    // 天气覆盖层
  }
}
```

**性能策略**：
- 使用 `OffscreenCanvas` 双缓冲
- 粒子系统用对象池（Object Pool），避免 GC
- 地形使用预渲染的 `ImageBitmap`，只在窗口 resize 时重绘
- 目标：60fps @ 1080p，CPU 占用 < 5%

### 3.5 前后端通信

```typescript
// api/wails.ts
// 绑定 Go 方法
export const ReadConfig = (tribe: string) => window.go.main.App.ReadConfig(tribe);
export const LaunchTribe = (tribe: string, cwd: string) => window.go.main.App.LaunchTribe(tribe, cwd);

// 订阅 Event
export function subscribeVital(callback: (vital: VitalSign) => void) {
  window.runtime.EventsOn('tribe:vital', callback);
}
export function subscribeForge(callback: (update: ForgeUpdate) => void) {
  window.runtime.EventsOn('forge:update', callback);
}
```

**事件流设计**：
- `tribe:vital` — 每 2s 推送生命体征（驱动呼吸动画）
- `tribe:born` — 进程启动（驱动图腾升起）
- `tribe:death` — 进程终止（驱动碎裂动画）
- `forge:update` — Token 消耗实时流（驱动熔炉液面）
- `fs:change` — 配置文件变更（驱动 DNA 重组）
- `eye:flash` — 灵动岛通知

## 五、扩展性设计

### 5.1 新 CLI 接入流程

1. **创建适配器**：`tribes/newcli_adapter.go`，实现 `TribeAdapter`
2. **注册**：`core/registry.go` 添加 `Register("newcli", NewNewCliAdapter())`
3. **前端地形**：`canvas/terrains/newcli.ts` 定义地形渲染参数（颜色、建筑类型、动画）
4. **配置 Schema**：`frontend/config-schemas/newcli.json` 定义配置项 UI 表现

**零代码扩展（未来）**：通过 JSON 描述文件定义新 CLI 的进程名、配置路径、日志格式，自动适配。
